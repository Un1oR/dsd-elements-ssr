# Fast Test Execution

## Когда применять
- Любой запуск автотестов: backend, frontend, e2e, утилиты.
- Перед пушем/PR, при локальной проверке изменений, в CI-репродукции.

## Правила
- Используй доступные ускорители: параллельный запуск, шардирование, кэш, инкрементальные режимы — только если они не скрывают фейлы.
- Для `pytest` запускай параллельно: `python -m pytest -n auto ...` (добавь свои флаги по месту).
- Не глуши сигнал: оставляй полноценный вывод ошибок, не скрывай капсулирующие команды без нужды.
- Если ускоритель нестабилен (флейк/гонка) — фиксируй или временно отключай и сообщай.

## Быстрый чек-лист
- Параллель? ✅
- Кэш/инкрементальный режим? ✅ если есть и надёжен.
- Полный вывод сохраняется? ✅
- Известные флейки помечены/починены? ✅
